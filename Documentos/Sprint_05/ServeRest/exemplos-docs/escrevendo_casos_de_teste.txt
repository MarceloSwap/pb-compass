# Escrevendo Cenários de teste na API SERVEREST

POST - Fazer login com sucesso na SERVEREST
- Dados Vaálidos
- Ter um usuário cadstrado
- A requisição deve voltar um status 201

GET - No endpoint /usuarios da aí ServeRest deve listar todos os usuários
- Mostrar todos os usuários cadastrados na aplicação
- A requisição deve voltar um satuts 200

POST - Criar um usuário novo dentro da api ServeRest em /usuarios
- Dados Vaálidos
- O usuário não deve estar cadastrados
- O usuário deve ter e-mail e senha
- A requisição dve voltar um status 201

POST - Fazer login com um usuário inválido na API ServeRest
- Utilizar dados inválidos
- Não pode estar cadastrados
- A requiciç~so deve voltar um status 400

POST - Tentar criar usuário com dados invalidos na API ServeRest em /usuários
- Utilizar dados inválidos no cadastro
- O usuário pode já estar cadastrado
- A requisição deve voltar um status 400

PUT - Tentar editar usuário com id "0uxuPY0cbmQhpEz1"
- Utilizar dados vaálidos para edição
- Preciso ter certeza que o usuário já está cadastrado
- A requisição deve retornar um status code 200

DELETE - Quero excluir usuário com id "0uxuPY0cbmQhpEz1"
- Validadar que o usuario com id existe
- Não é permitido excluir usuário com carrinho
- A requisição deve retornar um status code 200
